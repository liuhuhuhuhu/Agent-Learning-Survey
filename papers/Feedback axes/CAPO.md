## 1. The problem 

Current Reinforcement Learning with Verifiable Rewards (RLVR) assigns a single binary reward to an entire generated response, providing no differentiated feedback for intermediate reasoning steps. Existing fine-grained credit assignment methods either rely on unreliable value estimation or require expensive Process Reward Models (PRMs) with manually annotated supervision, making them inefficient and prone to reward hacking.


## 2. The core idea

CAPO (Credit Assignment Policy Optimization) introduces a simple yet effective credit assignment framework that uses an off-the-shelf LLM as a **Generative Process Reward Model (LLM-as-GenPRM)**. Instead of predicting process rewards with a learned reward model, the LLM directly critiques each reasoning trajectory in a single pass, identifies incorrect reasoning steps, and converts these critiques into deterministic token-level rewards for reinforcement learning.

## 3. How it works

- Roll out multiple reasoning trajectories using the policy model.
- Use a rule-based verifier to determine whether the final answer is correct.
- Prompt an off-the-shelf LLM (LLM-as-GenPRM) to identify incorrect reasoning steps in one inference pass.
- Aggregate multiple critiques through voting mechanisms (e.g., Majority Vote or Intersection Vote) to improve robustness.
- Convert incorrect steps into token-level penalties and combine them with outcome rewards using an asymmetric reward shaping strategy before RL optimization.



## 4. How they evaluate it

The authors evaluate CAPO on four mathematical reasoning benchmarks:

- MATH
- OlympiadBench
- AMC2023
- AIME2024

and three out-of-distribution reasoning benchmarks:

- GPQA-Diamond
- ARC-c
- MMLU-Pro

using Llama-3 (1B/3B) and Qwen2.5 (1.5B/7B) backbones.

Headline results:

- CAPO consistently outperforms GRPO, supervised fine-tuning, and reward-model-based RL methods.
- Demonstrates improved mathematical reasoning and better generalization on OOD reasoning tasks.
- Larger numbers of generated critiques further improve performance through voting.
- The proposed asymmetric reward shaping (Outcome > Process) provides the most stable learning dynamics.


## 5. Why it matters for OUR review

CAPO proposes a practical solution to one of the central problems in reinforcement learning for LLM agents: **fine-grained credit assignment without training an auxiliary reward model**.

Within our framework:

- **Adaptation:** improves policy learning through reliable token-level credit assignment.
- **Evolution:** iteratively refines policy behavior using process critiques generated by an external LLM.

Connections to related papers:

- **iStar (2025):** both address credit assignment, but iStar trains an implicit Process Reward Model, whereas CAPO directly uses an off-the-shelf LLM to generate process critiques.
- **HPL (2026):** HPL performs hierarchical preference optimization, while CAPO focuses on token-level reward shaping.
- **PRIME (2025):** PRIME predicts process rewards using learned reward models; CAPO avoids reward model training altogether.

---

## 6. Limitations / open questions 

CAPO depends on the quality of the external LLM used as the Generative Process Reward Model, making inference more expensive than standard RLVR. It also assumes that reasoning steps can be accurately segmented and critiqued by the verifier LLM, which may become challenging in open-ended or highly interactive agent environments.


## 7. One-sentence takeaway


CAPO replaces learned process reward models with an off-the-shelf LLM that directly generates deterministic step critiques, enabling reliable and efficient token-level credit assignment for reinforcement learning.
