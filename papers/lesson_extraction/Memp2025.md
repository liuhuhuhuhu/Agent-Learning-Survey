# 1. The problem 

Current LLM agents usually solve each task independently and repeatedly explore similar environments from scratch. Existing memory systems mostly store trajectories or prompts but do not systematically study how procedural memory should be built, retrieved, updated, and maintained over time. :contentReference[oaicite:1]{index=1}


# 2. The core idea 

The paper proposes **Memp**, a procedural memory framework that allows agents to learn reusable task-solving skills from previous trajectories. Instead of treating memory as static knowledge, Memp continuously builds, retrieves, updates, and removes procedural memories so that the memory repository evolves together with the agent's experience. :contentReference[oaicite:2]{index=2}


# 3. How it works 

- Build procedural memory by converting successful trajectories into reusable procedural knowledge (scripts and trajectory abstractions).
- Retrieve the most relevant procedural memory using semantic similarity before solving a new task.
- Execute the task while using retrieved procedural memory as guidance.
- Update the memory repository by adding new experience, correcting failed memories through reflection, and removing outdated or ineffective memories.
- Repeat this process continuously so that procedural memory gradually improves over time. :contentReference[oaicite:3]{index=3}


# 4. How they evaluate it

The authors evaluate Memp on **TravelPlanner** and **ALFWorld** using GPT-4o, Claude 3.5 Sonnet, and Qwen2.5-72B.

Compared with ReAct, Expel, and Agent Workflow Memory (AWM), Memp consistently achieves higher task success rates while requiring fewer execution steps. The experiments also show that procedural memory can be transferred from stronger models to weaker models and continues improving through online memory updates. :contentReference[oaicite:4]{index=4}


# 5. Why it matters for OUR review

This paper is one of the core papers in the **Evolution Layer** because it treats procedural memory as an evolving object instead of a static storage module. It connects memory management with continual learning by introducing explicit strategies for memory construction, retrieval, correction, and deletion.

It is closely related to **ReasoningBank** (reasoning memory), **ERL** (experience extraction), and **Reflexion** (reflection-based updating), but focuses more on maintaining an editable procedural memory repository.


# 6. Limitations / open questions 

Memory retrieval still mainly depends on vector similarity with manually designed retrieval keys, and the framework relies on benchmark reward signals for memory updates. Its effectiveness in open-ended real-world environments without reliable feedback remains unclear. :contentReference[oaicite:5]{index=5}


# 7. One-sentence takeaway

Memp shows that procedural memory should be treated as a continuously evolving system that can be built, retrieved, updated, and refined throughout an agent's lifetime.
