**Title:** SkillWeaver: Web Agents can Self-Improve by Discovering and Honing Skills

**Authors / Year / Venue:**
Boyuan Zheng, Michael Y. Fatemi, Xiaolong Jin, et al. / 2025 / arXiv

**Coder (you):**

**Framework layer(s):**

## 1. The problem
Existing web agents mainly improve by storing trajectories, using demonstrations, or fine-tuning model parameters. These approaches struggle to explicitly abstract reusable procedural knowledge, transfer learned behaviors across websites, and continually expand an agent’s capabilities without retraining.


## 2. The core idea

SkillWeaver enables web agents to autonomously explore websites, discover reusable skills, and convert successful interaction trajectories into executable Python APIs. These APIs are tested, debugged, accumulated in a growing skill library, and reused or composed into more complex skills, allowing continual self-improvement without updating model parameters.


## 3. How it works

- Automatically proposes novel website skills through autonomous exploration, including procedural, navigational, and information-seeking tasks.
- Executes proposed tasks and evaluates successful trajectories using an LLM-based reward model.
- Synthesizes successful trajectories into reusable Python APIs with function signatures, documentation, and usage logs.
- Performs automated unit testing, parameter generation, debugging, and refinement to improve API robustness.
- Stores synthesized APIs in a growing skill library, enabling future reuse and composition into increasingly complex skills.


## 4. How they evaluate it

The authors evaluate SkillWeaver on:

- **WebArena**
- **Online-Mind2Web (real-world websites)**

Headline results:

- Achieves **31.8% relative improvement** in success rate on WebArena after integrating synthesized APIs.
- Achieves **39.8% relative improvement** on real-world websites.
- APIs generated by stronger GPT-4o agents improve GPT-4o-mini by up to **54.3%**, demonstrating strong transferability.
- Synthesized APIs perform comparably to manually written APIs on websites with low or medium API support.


## 5. Why it matters for OUR review

SkillWeaver demonstrates that self-improving web agents can continually accumulate procedural knowledge as executable APIs rather than raw trajectories.

Within our framework:

- **Memory:** stores reusable APIs in a persistent skill library.
- **Adaptation:** discovers, synthesizes, tests, and debugs new APIs through autonomous exploration.
- **Evolution:** continuously expands the API repertoire and enables increasingly complex skill composition over time.

Connections to related papers:

- **Voyager (2023):** both convert exploration experiences into reusable executable skills, but SkillWeaver focuses on web environments and synthesizes browser automation APIs.
- **ALITA (2025):** both automatically generate reusable capabilities, although ALITA packages them as MCPs while SkillWeaver stores executable Python APIs.
- **MUSE-Autoskill (2026):** both maintain reusable skill repositories, but MUSE emphasizes lifecycle management, whereas SkillWeaver emphasizes autonomous website exploration and API synthesis.


## 6. Limitations / open questions

The quality of synthesized APIs depends heavily on the underlying LLM's ability to correctly generate code and identify appropriate API parameters. In addition, API selection and parameter prediction remain major bottlenecks, especially for weaker language models.


## 7. One-sentence takeaway

If a reader remembers only one thing about this paper, it should be:

Web agents can continually improve themselves by autonomously exploring websites, distilling successful interaction trajectories into reusable executable APIs, and composing those APIs into increasingly sophisticated skills without retraining the underlying language model.
